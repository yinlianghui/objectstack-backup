# Issue 2678 Minimal Evidence Design

## Objective

Create a small, durable evidence package that answers the maintainer's concrete questions about issue #2678's passed batching result: test environment, tested objects/components, input data, row and column counts, write counts, persisted values, and summary results.

This supplement does not recreate the deleted historical harness. It runs new, explicitly identified evidence fixtures against the exact historical QA commit `98874656ffc50ce1531af52346228ffcdda73fba`.

## Scope

The supplement executes only four scenarios:

1. Seed Loader inserts 1,000 rows in batches of 200.
2. Import Runner inserts 1,000 rows in batches of 200.
3. ObjectQL inserts 1,000 children for one parent in five batches.
4. ObjectQL inserts 1,000 children for ten parents in five batches.

It does not rerun confirmed issues #2804 or #2805, classifier boundary #2802, PR regressions #2803/#2806, UI, 50,000-row cancellation, Turso/libSQL, HotCRM, or the historical full suites.

## Environment and Components

- Framework commit: `98874656ffc50ce1531af52346228ffcdda73fba`
- Runtime: Node.js 22.14.0 and pnpm 10.31.0 on Apple Silicon
- Persistence: a fresh local better-sqlite3 database for each scenario
- Components: real `SeedLoaderService`, `runImport`, `ObjectStackProtocolImplementation`, `ObjectQL`, and `SqlDriver`
- Instrumentation: wrappers around real engine/driver methods; successful writes still delegate to the real implementation

## Plain Seed and Import Objects

`qa_seed_item` and `qa_import_item` use the same schema:

| Field | Type | Input or generated |
|---|---|---|
| `id` | text primary key | generated by the SQL driver |
| `external_key` | text | input |
| `name` | text | input |
| `amount` | number | input |
| `active` | boolean | input |

Each scenario therefore has 1,000 input rows × 4 input columns and five stored fields after generated IDs are included.
